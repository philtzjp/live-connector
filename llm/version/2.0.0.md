# 2.0.0

MCP endpoint の認証方式とインストール型運用手順を更新する major release。`LIVE_CONNECTOR_MCP_TOKEN` による Bearer 認証を廃止し、loopback Host / Origin header 検証へ切り替える。

## 破壊的変更

- `LIVE_CONNECTOR_MCP_TOKEN` は環境変数モデルから削除する。
- MCP client 設定で `Authorization` header は不要になる。
- `LIVE_CONNECTOR_MCP_HOST` は `127.0.0.1` / `localhost` / `::1` の loopback host のみ許可する。

## 動作要件

- Ableton Live（Extensions 対応の Beta ビルド）
- インストール型運用: Developer Mode OFF + `.ablx` インストール
- 開発モード: Developer Mode ON + `extensions-cli run`
- Extension Host 同梱の Node ランタイム（≥ 24.14.1）

## 実行・配布

- MCP トランスポートは Node 標準 `http` 上の `@modelcontextprotocol/sdk` `StreamableHTTPServerTransport`（ステートレス）。エンドポイント `/api/v1/mcp`、ヘルスチェック `/health`。
- `/api/v1/mcp` は loopback Host / Origin header 検証でローカル接続のみを受け付ける。
- 配布アーカイブは `pnpm package` で `apps/extension/dist/live-connector-2.0.0.ablx` に生成する。
- インストール型運用では `.ablx` を Ableton Live の Extensions ページにドロップし、Developer Mode OFF で Live 起動時に自動ロードする。
- Claude Code は `claude mcp add --transport http live-connector http://127.0.0.1:7799/api/v1/mcp --scope project` で登録する。MCP 設定変更時のみ Claude Code 再起動が必要である。

## 制約

- ネイティブプリセット（`.adv` / `.adg` / 第三者プラグインのプリセット）の読込は SDK 非対応。`search_presets` は探索のみ。デバイス状態の保存・再適用は host 公開パラメータに限定。
- 第三者プラグインの挿入は SDK 非対応（`insertDevice` は内蔵デバイスの既定のみ）。
