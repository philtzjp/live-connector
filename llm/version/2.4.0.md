# 2.4.0

読み取り応答のサイズを制御する minor release。

## 追加機能

- `query`: LIMIT を省略したクエリに既定行数上限 `DEFAULT_ROW_LIMIT`（500）を導入する。上限超過時は先頭 500 行に切り詰め、応答に `truncated: true` と絞り込み手段（LIMIT/SKIP・パターン/WHERE の絞り込み・集計）を示す `hint` を返す。LIMIT 明示時は評価器が適用済みのため素通しし、既存挙動を変えない。
- `get_overview`: 粒度パラメータを追加する。`includeClips`（既定 true、false で各トラックの `arrangementClips` 明細を省略し `arrangementClipCount` のみ残す）、`trackOffset` / `trackLimit`（トラック範囲の指定）。応答に `trackOffset` / `tracksShown` / `includeClips` を含める。`arrangementEndTime` と `trackCount` は範囲指定に関わらず全体を反映する。

## 変更内容

- 行数上限を `query.ts` の定数 `DEFAULT_ROW_LIMIT` に一元管理し、ツール説明に明記する。
- `llm/models.yaml` の `query` / `get_overview` 定義を更新する。

## 動作確認

- `pnpm test`: `applyRowCap`（上限未満は素通し・上限超過で切り詰めと hint・LIMIT 明示で不変）と `get_overview`（includeClips:false でクリップ明細省略・trackLimit で範囲制限）を検証。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- 切り詰めは行の materialize 後に行う（評価器は全束縛を射影してから SKIP/LIMIT を適用する既存構造）。真の early-stop は ORDER BY / 集計が全行を要するため導入しない。
- 完全なページングは SKIP（#48 実装済み）と併用する。
