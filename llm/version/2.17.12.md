# 2.17.12

get_render_job が実行中ジョブを破棄しないようにする patch release。

## 修正内容

- レンダリングジョブの保持（MAX_RENDER_JOBS=50）超過時の evict を done / error のジョブのみ（古い順）に変更する。従来は status を見ずに挿入順で evict していたため、51 件以上のジョブ生成で running が追い出され、get_render_job が not_found を返し続けていた（#55 の「結果が不定にならない」契約の破れ）。
- running がジョブ保持数の上限を占有している場合、新規 background 開始を hint 付き BadRequestError で拒否する（開始しても保持できず照会不能になるため）。
- テスト用に `clearRenderJobsForTest()` を追加（module singleton のリセット）。

## 動作確認

- `pnpm test`: running を残したまま完了ジョブ 55 件で上限を溢れさせても running が照会可能なこと・running 50 件占有時の新規 background 拒否を検証（audio 4 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
