# 2.14.0

MCP の initialize instructions とツールアノテーションを整備する minor release。

## 追加機能

- `createMcpServer` の initialize 応答に `instructions` を設定する。推奨手順（schema → query → preview → confirm）、時刻座標の 2 系統（クリップ相対 vs アレンジメント絶対）、ガードレール（confirm 閾値 20・query 500 行切り詰め・delete_* の confirm・snapshotId / restore_snapshot・batch の単一 undo）、ミキサー Parameter 経路の要約を含む。
- 全ツールに実態と一致した annotations を付与する。`withToolAnnotations` facade が `TOOL_ANNOTATIONS`（正本）に基づき注入する: read 系は `readOnlyHint`、`delete_*` は `destructiveHint`、`set_*` / `apply_device_state` / `restore_snapshot` は `idempotentHint`、生成系は非冪等。facade はツール名・ハンドラを変えないため `tools/list` と `describeRegisteredTools` に影響しない。
- `EXAMPLE_QUERIES` にミキサー Parameter の読み取り例（HAS_VOLUME / HAS_SEND）を、`query_contract.select.valid_examples` に `set_device_parameter` 用の HAS_PAN select 例を追加する。

## 設計判断（記録）

- ミキサー書き込み: `volume` / `panning` / `send` は `HAS_MIXER` → `Parameter` 経由で `set_device_parameter` に統一する。`set_track` へ直接プロパティを追加しない。理由: ミキサー値は min/max/quantized を持つ `Parameter` であり、`set_device_parameter` が一貫して扱えるため。
- MCP prompts: 本 release では採用しない。定型ワークフローの配布は `instructions` で代替する。採用する場合は別 Issue に分割する。

## 動作確認

- `pnpm test`: `withToolAnnotations`（read/delete/idempotent の注入・未知ツールは無注入）と `TOOL_ANNOTATIONS` を検証。
- `pnpm --filter @live-connector/extension build`: バンドルに instructions が注入される。`pnpm typecheck`: 全パッケージ成功。

## 制約

- annotations はヒントであり、クライアントの扱いは実装依存。read/write の実際の安全性はツール実装のガードレールが担保する。
