# 2.17.4

cypher 集計とパースエラーの仕様乖離（#48 実装の残課題）を修正する patch release。

## 修正内容

- 純集計クエリ（グルーピングキー無し）が空マッチで 0 行を返していたのを、1 行（count / sum = 0、min / max / avg = null）を返すよう修正する。「0 件」と「パターン不一致」の区別を不要にする Cypher 標準の挙動。グルーピングありの空マッチは従来どおり 0 行。
- MATCH で束縛されていない変数の集計（例: `count(x)`）が無言で 0 を返していたのを、WHERE / RETURN / ORDER BY と同じ hint 付き `Unknown variable` エラーにする。
- min / max が数値のみ集計し文字列プロパティで null を返していたのを、文字列も対象（数値同士は数値比較、それ以外は辞書順）にする。sum / avg は従来どおり数値のみ。
- パースエラーに対応済み文法の hint を付与する（AS / WITH / count(DISTINCT ...) / CREATE 系の非対応を明示）。

## 動作確認

- `pnpm test`: 空マッチ純集計の 1 行応答・グルーピングあり空マッチの 0 行・未知変数集計の拒否・min/max の文字列辞書順・パースエラー hint 3 種を検証（cypher 54 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
