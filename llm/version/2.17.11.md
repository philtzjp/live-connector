# 2.17.11

書き込み履歴の並び順と要約の情報落ちを修正する patch release。

## 修正内容

- `get_write_history` の entries を自己記述どおり新しい順で返す（従来は古い順。list_snapshots とも整合）。
- `summarizeResult` の allowlist に `snapshotId` / `undoSnapshotId` / `snapshots` を追加し、履歴エントリから restore_snapshot へ辿れるようにする。生成物のネスト応答（clip / track / device / scene / cuePoint / sample）は識別フィールド（index / name / kind / id）を保持し、batch の steps はステップ数とツール名の内訳（`{count, tools}`）を保持する。
- 履歴・スナップショットの fs 読み取りで ENOENT のみを「存在しない」として扱い、EACCES / EIO 等の storage 障害は空の正常応答で隠さずエラーにする（`isFileMissingError()` を共有。readSnapshot の not_found 判定・listSnapshots も同様）。

## 動作確認

- `pnpm test`: 新しい順の応答・snapshotId / ネスト識別 / batch 内訳の要約保持・EISDIR（ENOENT 以外の fs 障害の代表）でのエラー応答を検証（history 9 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
