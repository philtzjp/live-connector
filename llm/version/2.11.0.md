# 2.11.0

書き込み操作の履歴を記録し参照する手段を追加する minor release。

## 追加機能

- `get_write_history`: 書き込みツールの実行履歴（時刻・ツール名・入力要約・結果）を新しい順に取得する。`limit`（既定 50、最大 500）・`since` / `until`（ISO 時刻）で絞れる。件数上限超過時は `truncated:true` を返す。ホスト再起動をまたいで参照できる。
- `withWriteHistory` facade: `createMcpServer` の登録経路に挿入し、`WRITE_HISTORY_TOOLS` に含まれる書き込みツールのハンドラをラップする。結果が `status:"ok"` の実書き込みのみ `environment.storageDirectory/history/write-history.jsonl` へ JSONL 追記する（preview / confirm_required / error は記録しない）。ツール名は変えないため `tools/list` と `describeRegisteredTools` に影響しない。

## 変更内容

- 入力は要約して記録する（配列は件数、長い文字列は 200 文字で切り詰め）。結果は主要フィールド（status / modified / matched / noteCount / removed / added / affected / applied 等）のみ抽出する。
- ローテーション: ファイルサイズが `MAX_HISTORY_BYTES`(512KiB) を超えたら末尾 `MAX_HISTORY_ENTRIES`(1000) 件へ切り詰める。
- `llm/models.yaml` / `llm/ARCHITECTURE.md` を更新する。

## 動作確認

- `pnpm test`: `summarizeInput` / `summarizeResult` と、facade 経由の記録（ok のみ・preview 除外・write set 外は非記録）＋別 deps からの読み出し（ホスト再起動相当）を temp ディレクトリで検証。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- 記録は `status:"ok"` を返す書き込みツールに限る。将来 `status` を返さない書き込みが増えた場合は `WRITE_HISTORY_TOOLS` と記録条件の見直しが必要。
- 関連: #50（巻き戻し。スナップショット識別子を履歴に含めることで復旧導線が繋がる）。
