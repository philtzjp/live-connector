# 2.3.0

Cypher サブセットに集計・整列・重複排除・オフセットを追加する minor release。

## 追加機能

- RETURN の集計関数を実装する: `count(*)` / `count(var)` / `count(var.prop)` / `min` / `max` / `avg` / `sum`。非集計項目と併用すると、非集計項目をキーとした暗黙グルーピングで集計行を返す。`sum` は空集合で `0`、`min` / `max` / `avg` は空集合で `null`。
- `ORDER BY <expr> [ASC|DESC]`（カンマ区切りの多段キー）を実装する。非集計クエリでは RETURN に無いプロパティでも整列でき、集計クエリではグルーピングキーまたは集計を参照する。
- `RETURN DISTINCT` による行の重複排除を実装する。
- `SKIP <integer>` を実装し、`LIMIT` と組み合わせたページングを可能にする。`SKIP` / `LIMIT` は集計・整列後に適用する。

## 変更内容

- 文法: `MATCH <pattern> [WHERE <expr>] RETURN [DISTINCT] <items> [ORDER BY <expr> [ASC|DESC], ...] [SKIP <integer>] [LIMIT <integer>]`。
- `LIMIT` / `SKIP` に負値検証を追加する（非負整数のみ）。
- `schema` ツールの `query_contract`・`EXAMPLE_QUERIES`、`query` ツール説明、`llm/models.yaml` の cypher_contract を新文法に更新する。
- 書き込み系ツールの `select`（単一ノード変数 RETURN）の契約は変更しない（`SKIP` / `LIMIT` は select でも適用可能）。

## 動作確認

- `pnpm --filter @live-connector/cypher test`: parser（集計・ORDER BY・SKIP・エラー系）と evaluator（グルーピング・min/max/avg/sum・整列・DISTINCT・ページング）をフェイク GraphAdapter で検証（43 件）。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- ORDER BY はプロパティまたは集計を参照する。裸のノード変数（`ORDER BY t`）は hint 付きエラーとする。
- 集計クエリの ORDER BY は RETURN に現れるグルーピングキーまたは集計のみ参照できる。
