# 2.16.0

MIDI / Audio トラック生成ツール `create_track` を追加する minor release（SDK 実機能に基づく scope）。

## 追加機能

- `create_track`: `kind`（`midi` / `audio`）と任意の `name` を受け取り、`Song.createMidiTrack()` / `Song.createAudioTrack()` でトラックを生成する。生成後に `Track.name` で命名する。`preview` に対応する。応答に生成後の実 `index` / `name` / `kind` を返す。

## 変更内容

- 挿入位置（index）は受け付けない。SDK の `createMidiTrack()` / `createAudioTrack()` は index 引数を持たず、生成位置は「最後に選択されたトラックの直後、未選択なら末尾」に固定されるため（Issue #14 のコメントで確認済みの制約）。ツール説明にこの挙動を明記する。
- `create_track` に annotations（create write）を付与し、書き込み履歴（#57）の対象に含める。`llm/models.yaml` / `llm/ARCHITECTURE.md` を更新。

## 動作確認

- `pnpm test`: `create_track` の preview（未生成）と生成＋命名をフェイク SDK Song で検証。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- 生成後にトラックを並べ替える API は無いため、厳密な位置制御は不可。生成直後の MidiTrack は音源が無く発音しないため、`insert_device` と併用する。
- 関連: #10（クリップ生成）、#15（デバイス挿入）。
