# 2.5.0

ノート時刻のクリップ境界検証を追加する minor release。

## 追加機能

- `write_notes`: 各ノートの `startTime` がクリップ相対範囲 `[0, クリップ長)` に収まるかを検証する。範囲外のノートがある場合、既定で拒否し、2 座標系（クリップ相対拍 vs アレンジメント絶対拍）の説明と該当ノートの index / startTime を含む hint 付き構造化エラーを返す。境界外を意図的に許容する場合は `allowOutOfRange: true` を指定する。
- クリップ長は `Math.max(duration, loopEnd, endMarker)` を採用し、ループ・内容窓を含む上限で誤検出を避ける。
- `write_notes` の応答（preview / ok）に `clipLength` と `outOfRange`（範囲外ノート数）を含める。

## 変更内容

- `query_contract` に `time_coordinates`（`absolute` / `relative`）を追加し、`schema` ツール応答から 2 座標系の規約を参照できるようにする。`QueryContract` 型を更新する。
- `write_notes` ツール説明に座標系規約を明記する。

## 動作確認

- `pnpm test`: `findOutOfRangeNotes`（範囲内は空・境界以上と負値を検出）と `clipNoteLength`（3 属性の最大）を検証。
- `pnpm typecheck`: 全パッケージ成功。境界内ノートの正常系挙動は不変。

## 制約

- クリップ長は上限側（loop / 内容窓を含む最大）で判定するため、ループクリップの正当なノートを誤検出しない一方、境界のわずかな超過は許容側に倒れうる。厳密な相対座標の取り違え（絶対拍の混入）は検出できる。
- `create_arrangement_clip` の `loopSettings` や `trim_clip` / `move_clip` の絶対/相対検証の拡張は本 release の対象外（write_notes と schema 規約を優先）。
