# 2.17.17

構造操作ツールの応答と検証の細部を補完する patch release。

## 修正内容

- `duplicate_device` の応答に複製後の実 index を含める（`parent.devices` を handle で照合）。同名デバイスが 2 件になった直後でも index で一意に select できる。
- `create_clip` の audio 経路に load_sample と同等の事前検証（絶対パス・対応形式・存在確認）を追加する（`assertSampleFile()` を samples.ts に抽出して共有）。不正パスが hint 無しの SDK 生エラーになるのを防ぐ。
- 無効な引数の組み合わせを明示エラーにする: MidiTrack スロットへの `audioFilePath`、AudioTrack スロットへの `length`（従来は無言で無視）。
- confirm ガードの hint 文言「destructive, non-undoable operation」を「MCP からは undo できない（Live の Edit > Undo では戻せる）」趣旨に修正する。

## 動作確認

- `pnpm test`: duplicate_device の index 応答（元の直後に挿入）・無効な組み合わせ 2 種の拒否・相対パス拒否・存在しないファイルの not_found を検証（structure 7 件 / create 4 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
