# 2.8.0

SDK が提供する残りの構造操作をツール化する minor release。

## 追加機能

- シーン操作: `create_scene`（index 指定、省略で末尾）/ `delete_scene` / `duplicate_scene`。
- トラック操作: `delete_track` / `duplicate_track`（regular MidiTrack / AudioTrack のみ。return / main は不可）。
- デバイス操作: `delete_device` / `duplicate_device`（`device.parent` の Track または Chain に対して実行）。
- セッションクリップ削除: `delete_session_clip`（`ClipSlot.deleteClip()`。アレンジメントクリップは `delete_arrangement_clip`）。
- `create_clip` を AudioTrack 対応にする。AudioTrack の ClipSlot では `audioFilePath` で `ClipSlot.createAudioClip` を呼び、AudioClip を生成する（MidiTrack は従来どおり `length` で空 MidiClip）。

## 変更内容

- 破壊的操作（`delete_*`）は `preview` で対象を確認でき、`confirm:true` が無い場合は `confirm_required` を返す（`guardDestructive`）。
- 単一対象 select・型検証・hint 付きエラーを各ツールに実装する。
- `query_contract.select.tools` / `llm/models.yaml` / `llm/ARCHITECTURE.md` を更新する。

## 動作確認

- `pnpm test`: `guardDestructive`（preview / confirm_required / proceed / preview 優先）を検証。
- `pnpm typecheck`: 全パッケージ成功（`Track` / `Chain` 双方の delete/duplicate device、`ClipSlot.createAudioClip` を含む）。

## 制約

- 構造操作の実 Live 反映は実機検証に委ねる（ユニットテストはガードとフェイクに限定）。
- `duplicate_scene` / `duplicate_track` / `duplicate_device` は元の直後に挿入される（SDK 仕様）。挿入位置の指定はできない。
