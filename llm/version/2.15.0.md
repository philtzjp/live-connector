# 2.15.0

内蔵デバイスカタログの実機整合を一括検証する手段を追加する minor release。

## 追加機能

- `verify_device_catalog`: 内蔵デバイスカタログ（`BUILT_IN_DEVICE_CATALOG`）全項目を一時 MidiTrack へ順に `insertDevice` で挿入試行し、`{ total, insertable, failed, failedNames, results }` を返す。1 回の呼び出しで全項目の挿入可否一覧が得られる。
- 各デバイスは試行直後に削除し、一時トラックも最後に削除するため Set にトラック・デバイスの残留が無い（Live の undo 履歴には試行の記録が残る）。

## 変更内容

- 純粋関数 `summarizeCatalogResults` を抽出しテスト。`CATALOG_DEVICE_NAMES` を `devices.ts` から export。
- 検証結果と Live バージョンの記録先・更新手順を `llm/device-catalog.md` に定義する（失敗項目のカタログ除外／実ロード名調査の運用を含む）。
- `verify_device_catalog` に annotations（idempotent write）を付与。`llm/models.yaml` / `llm/ARCHITECTURE.md` を更新。

## 動作確認

- `pnpm test`: `summarizeCatalogResults`（insertable/failed の分離・failedNames）を検証。
- `pnpm typecheck`: 全パッケージ成功。挿入試行自体は Live 実機での実行が必要。

## 制約

- 実行は Live 実機（Extension Host 稼働）が前提。ユニットテストは集計ロジックに限定する。
- SDK に Browser（列挙）API が追加された場合、本検証はカタログ自動生成に置き換える。
