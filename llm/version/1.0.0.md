# 1.0.0

初回リリース。Ableton Live を Cypher 風クエリで操作する MCP サーバー（Ableton Extensions SDK ベース）。配布物は `.ablx`。

## 動作要件

- Ableton Live（Extensions 対応の Beta ビルド）+ Developer Mode 有効化
- Extension Host 同梱の Node ランタイム（≥ 24.14.1）

## MCP ツール

- 内省 / 読み取り: `schema` / `get_overview` / `query`（Cypher サブセット: MATCH / WHERE / RETURN / LIMIT、可変長リレーション、抽象ラベル）
- プロパティ書き込み（宣言的セレクション＋型付き SET、preview / confirm / withinTransaction）: `set_song` / `set_track` / `set_clip` / `set_scene` / `set_device_parameter` / `set_cue_point`
- 生成 / 削除: `create_clip` / `create_arrangement_clip` / `delete_arrangement_clip` / `create_cue_point` / `delete_cue_point`
- MIDI: `write_notes`
- オーディオ: `render_audio`
- プリセット / デバイス状態: `search_presets` / `save_device_state` / `apply_device_state`

## 実行・配布

- MCP トランスポートは Node 標準 `http` 上の `@modelcontextprotocol/sdk` `StreamableHTTPServerTransport`（ステートレス）。エンドポイント `/api/v1/mcp`、ヘルスチェック `/health`。
- `/api/v1/mcp` は loopback Host / Origin header 検証でローカル接続のみを受け付ける。
- 配布アーカイブは `pnpm package` で生成（`extensions-cli package`）。

## 制約

- ネイティブプリセット（`.adv` / `.adg` / 第三者プラグインのプリセット）の読込は SDK 非対応。`search_presets` は探索のみ。デバイス状態の保存・再適用は host 公開パラメータに限定。
- 第三者プラグインの挿入は SDK 非対応（`insertDevice` は内蔵デバイスの既定のみ）。
