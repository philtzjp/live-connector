# 2.17.10

restore_snapshot の対象照合と異常系を堅牢化する patch release。

## 修正内容

- スナップショット schema を 1.1 に更新し、取得時に対象の識別子（SDK `Handle.id` の文字列。`targetIdentities` / `targetIdentity`）を保存する。全 capture 経路（set_* / write_notes / transform_notes / batch / restore_snapshot 自身）が識別子を渡す。
- properties 復元の対象照合を配列 index から identity ベースに変更する。スナップショット後にトラック増減・並べ替えがあっても旧値を別ノードへ順ズレ適用しない。一致した対象のみ部分復元し、`missingFromSet` / `unmatchedNow` と note で申告する。identity が使えない環境（フェイク等）は件数一致時のみ index 照合し、不一致は hint 付きで拒否する。
- 保存済み `requiredLabel` を復元時にも検証し、select の再解決が別種ノードを返す場合は拒否する。
- 対象が全て消えた場合は `restored:0` の ok ではなく NotFoundError を返す（notes 復元の例外と異常系を統一）。notes 復元は対象 clip の identity 不一致も拒否する。
- スナップショットファイルの破損（JSON 不正）と schemaVersion 不一致を hint 付き BadRequestError にする（従来は internal_error / 無検証）。
- restore 実行前に照合済み対象の現在値を自動スナップショット（tool: `restore_snapshot`）し、応答の `undoSnapshotId` で誤 restore を巻き戻せるようにする。

## 動作確認

- `pnpm test`: トラック並べ替え後の identity 照合復元（順ズレ無し）・対象消滅時の部分復元と件数申告・全対象消滅の拒否・identity 無し件数不一致の拒否・requiredLabel 検証・破損ファイル / schemaVersion 不一致の hint 付きエラーを検証（snapshots 9 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
