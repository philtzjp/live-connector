# 2.17.13

query の明示 LIMIT による行数上限の迂回を制御する patch release。

## 修正内容

- 明示 LIMIT にも絶対上限 `MAX_ROW_LIMIT`（2000 行）を適用する。クランプ時は truncated:true と、絶対上限であること・SKIP ページング / WHERE 絞り込み / 集計への誘導を含む hint を返す。
- 採用判断と根拠を `llm/models.yaml`（query の decision_98）に記録する。行数上限の目的（MCP クライアントのコンテキスト保護）をクライアント自身の LIMIT で迂回できるのは目的に反するため。絶対上限は既定の 4 倍とし、2000 行以下の既存 LIMIT 付きクエリの挙動は不変（#60 受け入れ条件との両立）。
- ツール説明に絶対上限を明記する。

## 動作確認

- `pnpm test`: 明示 LIMIT が既定上限を超えて絶対上限まで通ること・絶対上限超過のクランプ（truncated:true + hint）を検証（query 4 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
