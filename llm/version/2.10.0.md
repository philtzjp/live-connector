# 2.10.0

接続先 Live Set の識別と構造変更の検知手段を追加する minor release。

## 追加機能

- `get_overview` に `identity`（`SetIdentity`: `storageDirectory` / `songHandle` / `sdkNote`）と `structureDigest`（構造フィンガープリント）を追加する。1 コールで接続先 Set の識別と構造変化を判定できる。
- `/health` に `structure`（`{ digest, songHandle }`）を追加する。認証なしエンドポイントのため非 PII のダイジェストと不透明な `songHandle` のみを載せ、`storageDirectory` は含めない。読み取り失敗時は `null`。
- 構造フィンガープリント `structureDigest` は、トラック数・トラック名・種別・アレンジ/セッションクリップ数・デバイス数・シーン/キュー数から djb2 8 桁 hex を導出する。前回値との比較で構造変更を 1 コールで判定できる。

## 変更内容

- 純粋関数 `structureDigest` / `collectSetFeatures` / `setIdentity` を `apps/extension/src/lom/fingerprint.ts` に実装する。tempo など可変プロパティはダイジェストに含めない。
- `llm/models.yaml`（get_overview / HealthResponse）と `llm/ARCHITECTURE.md` を更新する。

## 動作確認

- `pnpm test`: `structureDigest`（決定性・構造差異・リネーム検知）と `collectSetFeatures`（クリップ/デバイス数集計）、`get_overview` の identity/digest を検証。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- SDK は Live Set の名称・ファイルパスを公開しない（`Environment` は language / storageDirectory / tempDirectory のみ、`Song` / `Application` に name / path getter は無い）。識別は `songHandle`（ホスト再接続で変化）と構造ダイジェストに依存する。
- push 型の変更通知は SDK 非対応のため、検知はポーリング（`get_overview` / `/health` の再取得と比較）に限る。
