# 2.2.0

バージョン文字列を単一ソース化し、`/health` に稼働ホストのツール構成情報を追加する minor release。

## 追加機能

- `/health` 応答に `tools`（`RegisteredToolsSummary`: `count` / `digest` / `names`）を追加する。`describeRegisteredTools` が `createMcpServer` と同一の登録経路を辿ってツール名を収集するため、稼働ホストのツール構成を `/health` のみで外形確認できる。`digest` はソート済みツール名の djb2 8 桁 hex。
- ツールのバージョン注入用に `apps/extension/src/version.ts`（`SERVICE_VERSION`）を追加する。

## 変更内容

- バージョンの正本をリポジトリルートの `VERSION` に一本化する。`build.ts` が `manifest.json` / `package.json` のバージョン一致を検証し、`__LIVE_CONNECTOR_VERSION__` として esbuild `define` でバンドルへ注入する。不一致時はビルドを失敗させる。
- `http.ts`（`SERVICE_VERSION` リテラル）と `mcp.ts`（`new McpServer({ version })` リテラル）の手書き `"2.1.0"` を排除し、`version.ts` の `SERVICE_VERSION` を参照する。
- 注入が無い場合、`version.ts` は暗黙のフォールバックを使わず `ConfigError` を投げる。

## 動作確認

- `pnpm --filter @live-connector/extension build`: バンドルに注入バージョンが含まれ、手書きリテラルが残らないこと。
- `pnpm test`: `describeRegisteredTools` がツール名一覧・件数・安定ダイジェストを返すこと（フェイク SDK で検証）。
- `/health` が `application/health+json` で `version: "2.2.0"` と `tools` を返すこと。
- 配布アーカイブは `pnpm package` で `apps/extension/dist/live-connector-2.2.0.ablx` に生成すること。

## 制約

- `/health` は認証なしエンドポイントのため、追加情報はバージョンとツール名の範囲に留める。
