# 2.6.0

`write_notes` に merge と clear_range モードを追加する minor release。

## 追加機能

- `write_notes` の `mode` に `merge` と `clear_range` を追加する。
  - `merge`: 既存ノートを保持したまま入力ノートを追加する。同一 `pitch` かつ同一 `startTime` の衝突は入力側で置換する（重複ノートの積み上げを避ける）。応答に `added` / `previous` を含める。
  - `clear_range`: `range`（クリップ相対拍 `[start, end)`）に該当する既存ノートのみ削除する。応答に `removed` / `range` を含める。
- 既存ノートを再送せずにノートの追加・範囲削除ができる。

## 変更内容

- `notes` を任意（既定 `[]`）にする。`clear_range` では `notes` を無視し `range` を必須とする。
- 各モードで preview に対応し、適用後のノート数（`noteCount`）と影響範囲を返す。境界検証（#59）は replace / merge に適用する。
- `llm/models.yaml` と `write_notes` ツール説明を更新する。

## 動作確認

- `pnpm test`: `mergeNotes`（追加・衝突置換・非衝突）と `clearNotesInRange`（範囲削除・排他境界）を検証。
- `pnpm typecheck`: 全パッケージ成功。`replace` の既存挙動は不変。

## 制約

- 衝突判定は `pitch` と `startTime` の完全一致。同一 pitch でも startTime が異なれば別ノートとして共存する。
