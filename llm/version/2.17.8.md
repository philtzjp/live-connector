# 2.17.8

batch の入力検証と失敗時挙動を単体ツールと整合させる patch release。

## 修正内容

- ステップの set 検証を緩い `z.record` から単体ツールと同一の per-property zod スキーマに変更する。`SET_TOOL_SET_SCHEMAS`（write.ts に正本を export）を単体 set_* と batch の両方が共有し、`set_song {tempo:-10}` や `set_clip {warpMode:"Bogus"}` が batch でも MCP 層（resolve 前）で拒否される。
- write_notes ステップの検証・変換計画を単体 write_notes と同じ `planNoteWrite()`（notes.ts に抽出）に統一する。空 notes の拒否・range 検証（逆転 / 完全範囲外）・tailOverflow 申告が batch でも等価になる。
- 同一クリップへの複数 write_notes ステップが resolve 時点の notes で事前計算され先行ステップの結果を消していた問題を、適用時の逐次再解決（`computeNextNotes()` がその時点の clip.notes から再計算）で解消する。
- 適用段階の失敗（SDK トランザクションはロールバックしない）で、ステップごとの成否を追跡し、応答の `appliedSteps` / `failedSteps` / `unappliedSteps` と `phase:"apply"` で部分適用を識別できるようにする。部分適用が残った失敗は書き込み履歴にも記録する（recordWrite を拡張）。
- 解決段階の失敗応答に `phase:"resolve"` を追加する。
- `llm/models.yaml` / `llm/version/2.13.0.md` / `llm/ARCHITECTURE.md` の「all-or-nothing」「入力検証は単体ツールと同一」の記述を実装に一致させる。

## 動作確認

- `pnpm test`: スキーマ等価性（tempo 負値 / warpMode 不正 / value 型不正の拒否）・適用段階失敗の注入（tempo setter が throw → failedSteps=[0]、appliedSteps=[1]、部分適用の残存確認）・同一クリップ 2 段 merge の両立・clear_range 逆転の resolve 拒否を検証（batch 10 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
