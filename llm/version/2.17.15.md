# 2.17.15

ツールアノテーションの destructive / idempotent 区分を実態に合わせる patch release。

## 修正内容

- MCP 仕様の `destructiveHint:false` は「additive な更新のみ」を意味するため、既存内容の置換・削除を行うツールを destructive に修正する: `write_notes`（replace は全置換・clear_range は削除）/ `transform_notes`（全置換）/ `batch`（両者を内包可能）/ `restore_snapshot`（スナップショット後の変更を上書き）/ `load_sample`（replaceSample = 置換）。
- `restore_snapshot` と `load_sample` は「置換書き込み」（destructive かつ idempotent）の新区分 `REPLACING_WRITE` とする。
- `move_clip` / `trim_clip` に `idempotentHint: true` を付与する（絶対指定のため再実行同結果）。
- `TOOL_ANNOTATIONS` と `registerAllTools()` の登録ツール名の完全一致テストを追加し、無注釈ツールの黙認と陳腐化したエントリの両方を検出する。

## 動作確認

- `pnpm test`: 置換系ツールの destructive 申告・idempotent 付与・網羅一致を検証（annotations 4 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
