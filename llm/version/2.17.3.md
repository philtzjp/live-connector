# 2.17.3

書き込み select で ORDER BY が無言で無視される回帰（PR #66 以降）を修正する patch release。

## 修正内容

- `selectNodes()`（`packages/cypher/src/evaluator.ts`）が ORDER BY を評価するようにする。適用順は identity 重複排除 → ORDER BY 並べ替え → SKIP / LIMIT。`MATCH (t:Track) RETURN t ORDER BY t.name LIMIT 1` の select が整列後の先頭ノードを返す。
- 書き込み select は identity で重複排除するため、RETURN 変数以外を参照する ORDER BY（他変数のプロパティ・集計・裸変数）は対象ノードごとに並べ替えキーが一意に定まらない。これらは hint 付き `BadRequestError` で拒否する。
- 契約を `packages/lom-schema/src/schema.ts` の `query_contract.select.return_contract` と `llm/models.yaml` の `write_select_return_contract.order_by` に明記する。

## 動作確認

- `pnpm test`: ORDER BY + LIMIT 1 の select が整列後の先頭を返す・DESC + SKIP の並べ替え・他変数 ORDER BY と裸変数 ORDER BY の拒否を検証（cypher 47 件全緑）。
- `pnpm typecheck`: 全パッケージ成功。
