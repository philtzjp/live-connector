# 2.17.2

songHandle の bigint 直列化障害を修正する patch release。

## 修正内容

- SDK の `Handle.id` は `bigint` で `JSON.stringify` が TypeError を投げるため、`setIdentity()` と `/health` の `structure.songHandle` を `String(handle.id)` で文字列化する（`serializeSongHandle()` を `lom/fingerprint.ts` に追加）。修正前は実機で `get_overview` が常時 internal_error、`/health` が writeHead 後の stringify 失敗で接続破棄（socket hang up）になっていた。
- `/health` のペイロード構築を `buildHealthPayload()`（`server/http.ts`）へ抽出し、フェイク SDK でテスト可能にする。
- `SetIdentity.songHandle` の型を `unknown` から `string` に変更（応答形状は `llm/models.yaml` に反映済み）。

## 動作確認

- `pnpm test`: フェイクの song handle を実機同等の `{ id: bigint }` に変更し、`get_overview` の identity（`songHandle: "7"`）と `buildHealthPayload` の JSON 直列化（`songHandle: "42"`、`JSON.stringify` が throw しない）を検証。song 読み取り失敗時に `structure: null` とログ記録も検証。
- `pnpm typecheck`: 全パッケージ成功。

- 関連: #86 / #51 / #62。実機での `/health` 200 応答確認は実機スモーク検証（#107）で行う。
