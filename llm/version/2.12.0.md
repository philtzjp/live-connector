# 2.12.0

書き込みを巻き戻す手段（スナップショット）を追加する minor release。

## 追加機能

- `set_*` / `write_notes` は適用直前に旧値を `environment.storageDirectory/snapshots/<id>.json` へスナップショットし、応答に `snapshotId` を返す。set_* は書き込むプロパティの旧値（対象ノードの serialize）を、write_notes は旧 notes を記録する。
- `restore_snapshot`: `snapshotId` を指定して変更前の値へ書き戻す。`select` を再解決して旧プロパティ値 / 旧 notes を適用する。
- `list_snapshots`: 保存済みスナップショット（id / 時刻 / ツール / 種別 / select）を新しい順に取得する。
- `restore_snapshot` は書き込み履歴（#57）にも記録される。

## 変更内容

- スナップショットは最新 `MAX_SNAPSHOTS`(100) 件でローテーションする。
- `restore_snapshot` の復元は best-effort。対象が削除・移動されている場合や `select` が異なる件数にマッチする場合は部分的・不可となり、`matchedNow` / `snapshotTargets` / `note` で明示する。
- `llm/models.yaml` / `llm/ARCHITECTURE.md` を更新し、upstream（Ableton Extensions SDK）への undo / redo API 追加要望を本機構の前提として記録する。

## 動作確認

- `pnpm test`: スナップショットの capture / list（新しい順・件数）と、未知 id の `restore_snapshot` が `not_found` を返すことを temp ディレクトリで検証。
- `pnpm typecheck`: 全パッケージ成功。

## 制約

- 対象範囲: `set_*` / `write_notes` は旧値復元が可能。構造操作（create / delete / duplicate）や `move_clip` / `trim_clip` の非可逆属性（warp グリッド・MIDI marker）はこの機構の対象外。
- SDK に undo / redo 実行 API が無いため、本機構は select 再解決による代替であり、Live 上の任意編集との整合は保証しない。
